# PRD: 이랜드 유통 매출 데이터 대시보드 서비스

**작성일:** 2026-04-28  
**작성자:** CSO실  
**버전:** v0.1 (Draft)  
**기술 스택:** Next.js · React · Supabase · Vercel

---

## 1. 개요 (Overview)

### 1.1 배경

CSO실은 매일 지점별·부문별 매출 데이터를 수작업으로 엑셀 정리 후 분석하고 있음. 이 과정에서 데이터 가공 시간 낭비, 인적 오류 발생, 인사이트 도출 지연 등의 문제가 반복됨.

### 1.2 목표

CSV/Excel 파일을 업로드하는 것만으로 매출 분석·시각화·인사이트 요약까지 자동으로 처리되는 내부 BI 대시보드를 구축한다.

### 1.3 범위 (Scope)

| 포함 | 제외 |
|------|------|
| 파일 업로드 및 DB 누적 저장 | 로그인 / 사용자 권한 관리 (v1 제외) |
| 지점·부문·브랜드별 매출 분석 | 외부 ERP/POS 직접 연동 (v1 제외) |
| 성장률·달성률 자동 계산 | 모바일 반응형 최적화 (v1 제외) |
| 매출 추이 그래프 | AI 예측 기능 (v1 제외) |
| 인사이트 자동 요약 | |
| CSV 다운로드·필터·초기화 | |

## 2. 사용자 및 사용 시나리오

### 2.1 주요 사용자

- **CSO실 담당자** — 매일 데이터 업로드 및 리포트 확인
- **경영진 / 본부장** — 대시보드로 KPI 현황 파악 (읽기 전용)

### 2.2 핵심 사용 흐름

```
1. 담당자가 당일 매출 CSV/Excel 파일을 준비
2. 대시보드 접속 → 파일 업로드 영역에 드래그&드롭 또는 파일 선택
3. 시스템이 파일을 파싱 → Supabase DB에 누적 저장
4. 대시보드가 자동 갱신:
   - 지점·부문·브랜드별 KPI 카드 표시
   - 매출 순위 차트 업데이트
   - 추이 그래프 렌더링
   - 인사이트 요약 문구 생성
5. 담당자가 날짜·브랜드·채널 필터 적용 후 결과 검토
6. 필요 시 CSV 다운로드 또는 결과 복사 버튼 사용
```

## 3. 데이터 모델 (가정 기반 — 확인 필요)

### 3.1 업로드 파일 예상 컬럼

| 컬럼명 | 타입 | 설명 | 필수 |
|--------|------|------|------|
| `store_code` | string | 지점 코드 | ✅ |
| `store_name` | string | 지점명 | ✅ |
| `division` | string | 부문 (예: 패션, 식품, 리빙) | ✅ |
| `brand` | string | 브랜드명 | ✅ |
| `channel` | string | 채널 (예: 오프라인, 온라인) | ✅ |
| `sale_date` | date | 매출 발생 일자 (YYYY-MM-DD) | ✅ |
| `sales_amount` | number | 매출액 (원) | ✅ |
| `target_amount` | number | 목표 매출액 (원) | ✅ |
| `margin_amount` | number | 마진액 (원) | 권장 |
| `ly_sales_amount` | number | 전년 동기 매출액 (원) | 권장 |

> **확인 필요:** 실제 컬럼명, 전년 데이터 포함 여부, 마진율/마진액 포함 여부 → 섹션 7 질문 참고

### 3.2 Supabase 테이블 구조 (초안)

**`sales_records` 테이블**
```sql
id              uuid PRIMARY KEY DEFAULT gen_random_uuid()
store_code      text NOT NULL
store_name      text NOT NULL
division        text NOT NULL
brand           text NOT NULL
channel         text
sale_date       date NOT NULL
sales_amount    numeric NOT NULL
target_amount   numeric
margin_amount   numeric
ly_sales_amount numeric
uploaded_at     timestamptz DEFAULT now()
file_name       text        -- 업로드 파일 추적용
```

**`upload_logs` 테이블**
```sql
id          uuid PRIMARY KEY DEFAULT gen_random_uuid()
file_name   text NOT NULL
row_count   integer
uploaded_at timestamptz DEFAULT now()
status      text  -- 'success' | 'error'
error_msg   text
```

## 4. 기능 요구사항 (Functional Requirements)

### 4.1 파일 업로드 & 파싱

| ID | 요구사항 |
|----|----------|
| F-01 | CSV, xlsx 파일 업로드 지원 (드래그&드롭 + 파일 선택 버튼) |
| F-02 | 업로드 시 컬럼 헤더 자동 매핑 및 유효성 검사 (필수 컬럼 누락 시 오류 표시) |
| F-03 | 중복 데이터 처리: 동일 `store_code + sale_date + brand` 조합은 upsert |
| F-04 | 업로드 진행률 표시 및 성공/실패 결과 알림 |
| F-05 | 업로드 이력(파일명, 업로드 일시, 행 수) 확인 가능 |

### 4.2 핵심 KPI 계산

| ID | 지표 | 산식 |
|----|------|------|
| F-06 | 총 매출 | `SUM(sales_amount)` |
| F-07 | 목표 달성률 | `SUM(sales_amount) / SUM(target_amount) × 100` |
| F-08 | 전년 대비 성장률 | `(당기 매출 - 전년 매출) / 전년 매출 × 100` |
| F-09 | 마진율 | `SUM(margin_amount) / SUM(sales_amount) × 100` |
| F-10 | 순위 | 매출액 기준 내림차순 (지점·부문·브랜드별 각각 산출) |

### 4.3 시각화

| ID | 차트 | 설명 |
|----|------|------|
| F-11 | KPI 카드 | 총 매출·달성률·성장률·마진율 요약 (상단 고정) |
| F-12 | 매출 추이 라인 차트 | 일별/주별/월별 전환 가능, 전년 동기 비교선 포함 |
| F-13 | 지점별 매출 바 차트 | 수평 바, 달성률 오버레이 표시 |
| F-14 | 부문별 매출 도넛 차트 | 비율 시각화 |
| F-15 | 브랜드별 매출 순위 테이블 | 순위·매출·성장률·달성률 컬럼, 정렬 가능 |
| F-16 | 성과 우수/저조 하이라이트 | 상위 3 / 하위 3 지점·부문·브랜드를 색상 배지로 표시 |

### 4.4 인사이트 요약

| ID | 요구사항 |
|----|----------|
| F-17 | 선택 기간 기준 자동 인사이트 텍스트 생성 (예: "○○ 지점이 이번 주 성장률 1위, △△ 브랜드는 목표 대비 -15% 미달") |
| F-18 | 인사이트는 대시보드 하단 또는 사이드 패널에 카드 형태로 표시 |
| F-19 | 결과 복사 버튼으로 인사이트 텍스트를 클립보드에 복사 가능 |

### 4.5 필터 & 데이터 제어

| ID | 요구사항 |
|----|----------|
| F-20 | 날짜 범위 필터 (달력 피커, 기본값: 최근 30일) |
| F-21 | 브랜드 멀티셀렉트 필터 |
| F-22 | 채널 필터 (전체 / 오프라인 / 온라인) |
| F-23 | 지점 필터 |
| F-24 | 부문 필터 |
| F-25 | 필터 초기화 버튼 (전체 필터 리셋) |
| F-26 | 필터 적용 상태가 URL 쿼리 파라미터에 반영 (공유 가능) |

### 4.6 데이터 내보내기

| ID | 요구사항 |
|----|----------|
| F-27 | 현재 필터 적용 데이터를 CSV로 다운로드 |
| F-28 | 다운로드 파일명: `sales_export_YYYYMMDD_HHmm.csv` |

### 6.2 디자인 원칙

- **색상:** 화이트 배경 + 다크 네이비 텍스트, 포인트 컬러 1개 (이랜드 브랜드 컬러 검토)
- **서체:** 시스템 폰트 우선 (Pretendard 권장)
- **차트 라이브러리:** Recharts 또는 Nivo
- **성과 지표 색상:** 달성률 100% 이상 → 파랑/그린, 미달 → 레드/오렌지
- **테이블:** 줄무늬(zebra stripe), 정렬 가능, 페이지네이션 없이 스크롤

## 7. 기술 아키텍처

```
[사용자 브라우저]
      │
      ▼
[Next.js App (Vercel)]
  ├── /app/page.tsx          — 대시보드 메인
  ├── /app/api/upload        — 파일 파싱 API Route
  ├── /app/api/sales         — 집계 쿼리 API Route
  └── /components/           — 차트·필터·테이블 컴포넌트
      │
      ▼
[Supabase]
  ├── PostgreSQL DB (sales_records, upload_logs)
  └── Storage (원본 파일 보관 — 선택)
```

### 기술 선택 근거

| 기술 | 이유 |
|------|------|
| Next.js App Router | SSR + API Route 통합, Vercel 최적화 |
| Supabase | PostgreSQL 기반, 관리 콘솔 내장, 무료 티어로 시작 가능 |
| React Query (TanStack) | 필터 변경 시 캐시 기반 빠른 갱신 |
| Recharts | React 친화적, 커스터마이즈 용이 |
| xlsx / papaparse | 클라이언트 사이드 파일 파싱 |

## 9. 미결 사항 및 추가 확인 필요 질문

아래 항목들은 PRD를 확정하고 개발을 시작하기 전에 답변이 필요합니다.

---

### [데이터 구조]

**Q1. 실제 엑셀 파일의 컬럼명을 알 수 있나요?**  
→ 예: `지점코드`, `지점명`, `부문`, `브랜드`, `매출액` 등 실제 헤더명 공유 부탁드립니다.  
→ 샘플 파일 5~10행만 익명 처리 후 공유해 주시면 더욱 정확하게 설계할 수 있습니다.

**Q2. 전년 동기 매출(LY) 데이터는 파일 안에 컬럼으로 포함되어 있나요?**  
→ 아니면 별도 파일로 관리하거나 DB에 쌓인 전년 데이터를 자동 참조해야 하나요?

**Q3. 목표(Target) 데이터는 어디서 오나요?**  
→ 매일 업로드하는 파일에 포함? 별도 월간/연간 목표 파일을 따로 업로드?

**Q4. 마진 데이터는 포함되어 있나요?**  
→ 마진액 / 마진율 중 어떤 형태로 관리 중인가요?

**Q5. 하나의 파일에 며칠치 데이터가 담기나요?**  
→ 예: 당일 1일치만? 주간? 월간 누적?

---

### [비즈니스 로직]

**Q6. 성장률 계산 기준 기간은?**  
→ 전년 동기 대비 (YoY)? 전월 대비 (MoM)? 전주 대비 (WoW)? 모두 필요?

**Q7. '성과 우수/저조' 기준은 어떻게 정의하나요?**  
→ 예: 달성률 110% 이상 = 우수, 85% 미만 = 저조 / 또는 상위·하위 N개?

**Q8. 채널(channel) 구분 값은 어떻게 되나요?**  
→ 예: 오프라인, 온라인, 면세, 기타 등 — 실제 사용 중인 분류 기준

**Q9. 지점 수 및 브랜드 수는 대략 몇 개인가요?**  
→ 대용량 데이터 처리 전략 및 필터 UI 설계에 영향

---

### [운영]

**Q10. 데이터 업로드 주기는?**  
→ 매일 1회? 특정 요일? 실시간?

**Q11. 동일 날짜 데이터를 재업로드하는 경우가 있나요?**  
→ 수정된 데이터로 덮어쓰기가 필요한지, 이력을 쌓아야 하는지

**Q12. 대시보드를 사용할 인원 및 사용 환경은?**  
→ 인원 수, 사내망 접속 또는 외부 접속 여부 (보안 검토 목적)
